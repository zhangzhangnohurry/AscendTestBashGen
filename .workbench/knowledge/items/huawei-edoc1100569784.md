## 测试脚本/命令生成相关规则

- `hccn_tool` 在驱动安装过程中默认安装；安装完成后通过软链接放在 `/usr/bin/`，实际路径为 `/usr/local/Ascend/driver/tools`。
- 容器内使用时，需要将宿主机 `/usr/bin/hccn_tool` 挂载到容器中。
- 工具不支持多线程并发操作；生成自动化脚本时应串行执行 `hccn_tool` 命令，避免并发调用。
- 工具无昇腾虚拟化实例场景；不要面向虚拟化实例生成命令。
- 命令模式：`-g` 为查询类命令，`-s` 为设置类命令，`-t` 为测试类命令。
- 单个 NPU 设备操作通式：`hccn_tool -mode -cmd -i <dev_id> [-options]`。`dev_id` 来自 `npu-smi info` 输出的 `NPU ID`。
- NPU 端口操作通式：`hccn_tool -mode -cmd -i <dev_id> -u <udie_id> -p <port_id> [-options]`。先执行 `hccn_tool -g -dev_info -i <dev_id>` 获取 `UDie ID` 和 `Port ID`。
- 网络设备操作通式：`hccn_tool -mode -cmd -i <dev_id> -d <dev_name> [-options]`。先执行 `hccn_tool -g -dev_info -i <dev_id>` 获取 `Interface Name/Name`。
- 对 Atlas 350 加速卡：单卡时无有效端口；多卡时通过 UB 总线互连。2 卡互连时每张卡端口取值为 `4、5、6、8`；4 卡互连时每张卡端口取值为 `4、5、6`。
- 大量端口类命令的 `port_id` 说明均引用 `查询NPU设备信息` 输出中的 `Port ID`，脚本应先发现设备/端口再生成端口命令。
- `-time <time_val>` 在实时带宽查询中取值范围为 `[100, 10000]` ms，默认 `1000`；在峰值带宽查询中取值范围为 `[1, 226]` ms，默认 `200`。
- 历史 link up/down 状态最多查询当前时间之前 10 次记录，回显按时间由近到远。
- `导出端口的寄存器信息` 前可先开启 Device 侧 debug 日志；执行后可用 `msnpureport -f` 收集 NPU 侧日志，在生成日志目录的 `/slog/device-os-X/debug/device-os` 或 `/slog/device-os-X/debug/device-X` 查看目标寄存器信息。
- 设置/清除/恢复/TLS/误码模式等 `-s`、`-t` 命令会改变设备状态或清除统计，测试脚本应默认标为有副作用并显式隔离执行。

### 替换NPU设备的TLS证书 (`j010`)

- 所属章节：设置类命令
- 功能：替换Host侧或Device侧的TLS证书信息。

- 初始证书仅用于部署阶段为设备接入用户网络建立初始安全通道，华为不对初始证书的安全性做承诺与保证。

- 对于将华为初始证书作为业务证书使用而导致的安全风险和安全事件，由用户自行处置并承担后果。

- 华为初始证书有效期自生产制造之日起计算，因设备生产批次不同，有效期截止到2099年。

- 建议用户通过部署PKI系统对现网设备、软件签发证书并做好证书的生命周期管理（为保证安全性推荐使用短有效期的证书）。

- 初始证书过期后，使用初始证书的业务会中断。
- 注意事项：- 预置（首次）通常在装备阶段完成，用户可使用该命令替换已存在的证书（自己生成，或从设备厂商或第三方机构获取）。建议用户运行业务前替换装备阶段的初始证书。

- 证书、私钥和吊销列表配置：

- 证书的级数是可选择的，最少二级（只包含设备证书、根证书），最多十五级（设备证书、根证书、二级根证书…十四级根证书），出厂默认使用三级（设备证书、根证书、二级根证书）。设备证书中建议带有唯一性信息，保障一机一密。

- 用户可以选择只替换证书，或单独更新吊销列表，但需要保证TLS证书和吊销列表是配套的；单独更新吊销列表的操作只支持对Device侧进行。

- 文件名长度：最多255Byte，包括后缀（如：pub.pem，文件名长度为7）。

- 文件大小：

- 证书：Device侧最大3072Byte，Host侧最大2048Byte。

- 私钥：最大4096Byte。

- 吊销列表：最大20480Byte。

- 证书签发者名称：最大长度为64Byte。

- 证书必须是X.509v3格式；根CA和中级CA证书的“基本限制”扩展域必须标明为“CA”，“密钥用法”扩展域中必须包含“证书签名”，所有数字证书和吊销列表必须处于有效期内，建议定期更新配置。

- 仅支持pem类型的证书和私钥，crl类型的吊销列表。

- 仅支持RSA私钥，RSA长度为[2048,4096]，私钥必须是密文的，且私钥的加密算法仅支持AES-256。

- 用户需确保私钥、证书和吊销列表配套，否则可能导致替换失败。

- 私钥和设备证书需要配套替换。

- 用户需要指定完整的证书链（-ca1到-caN，N越大，级别越低），需保证低级别证书是由高级别证书签发、设备证书和吊销列表由最低级别的根证书签发，否则会导致证书链校验失败及配置失败。

- 预置或者更新私钥时，输入的口令必须和生成私钥时输入的口令一致。

- 口令使用：

- 口令使用交互式命令行输入，且不回显，口令的长度为8~15，且最少包含大写字母、小写字母、数字、特殊符号中的2种组合。

- 输入的口令不能在弱口令字典中，否则会配置失败。弱口令字典在“/etc”（root用户）或者“/home/${USER}”（非root用户）目录下，文件名为“hccn_weak_dict.conf”，权限为600。用户可增删该文件内容，但不能重命名或者删除该文件，否则弱口令字典校验功能失效。

- Host侧证书配置：

- 当证书配置在Host侧时，配置的有效性与设备ID值无关。

- 配置Host侧的TLS证书时，需要保证hccn_tool文件的权限为555，其所在路径“/usr/local/Ascend/driver/tools”（指工具的安装路径）的权限为755。

- 配置完成后，要将密文的私钥删除，回显会提示“[WARNING] Please delete the private key file after updating for security considerations.”。

- 集合通信时会使用该命令配置的TLS证书套件进行TLS认证，若TLS认证失败，则会记录日志。
- 命令格式：
```bash
hccn_tool -s -tls -i <dev_id> -path <file path> -pri <filename> -pub <filename> -ca1 <filename> [-ca2 <filename>] ...[-ca14 <filename>] [-crl <filename>] [-host]
```
- 关键参数：
  - -i <dev_id> | 必选 | 软件的逻辑ID。执行npu-smi info命令查询到的“NPU ID”字段值即为-i参数取值。
  - -path <file path> | 必选 | 证书文件存放的系统位置路径。
  - -pri <filename> | 可选 | 私钥文件名。
  - -pub <filename> | 可选 | 设备证书文件名。
  - -ca1 <filename> | 可选 | 根证书（第1级）证书文件名称。
  - -caN <filename> | 可选 | N级根证书（第N级，1<N<15）证书文件名称。
  - -crl <filename> | 可选 | 吊销列表文件名称。
- 示例：
```bash
hccn_tool -s -tls -i 0 -path ./CA/level3_CA/2048/sha256/1 -pri server_2.key -pub server_2.pem -ca1 1thCA.pem -ca2 2_1thCA.pem
```
```bash
hccn_tool -s -tls -i 0 -path ./CA/level3_CA/2048/sha256/1 -crl cert.crl
```

### 设置端口的SU误码模式 (`j018`)

- 所属章节：测试类命令
- 功能：设置端口SU误码模式。
- 注意事项：- 该命令设置成功后，可执行查询端口的SU误码模式和误码统计结果命令查询设置的误码模式。

- 设置SU误码模式时，需保证配置时序，应先配置一个设备端口为发送使能，再配置互连的另一设备端口为接收使能。设置完成后，误码统计结果在接收端口通过查询端口的SU误码模式和误码统计结果命令查询。
- 命令格式：
```bash
hccn_tool -t -su_ber -i <dev_id> -u <udie_id> -p <port_id> -mode <mode_type>
```
- 关键参数：
  - -i <dev_id> | 必选 | 软件的逻辑ID。执行npu-smi info命令查询到的“NPU ID”字段值即为-i参数取值。
  - -u <udie_id> | 必选 | 芯片UDie ID。取值：0。可通过查询NPU设备信息命令查询。
  - -p <port_id> | 必选 | 芯片端口ID，详见查询NPU设备信息命令输出说明中的“Port ID”。
  - -mode <mode_type> | 必选 | SU误码模式：

- tx_enable：发送使能。

- rx_enable：接收使能。

- stop：不使能。未配置时，默认为stop。
- 示例：
```bash
hccn_tool -t -su_ber -i 0 -u 0 -p 4 -mode rx_enable
```

### 查询指定NPU设备到目的地址的UB ping结果 (`j019`)

- 所属章节：测试类命令
- 功能：查询指定NPU设备到目的地址的UB ping结果。
- 注意事项：Device侧支持的并发ping操作不大于8个。
- 命令格式：
```bash
hccn_tool -t -ub_ping -i <dev_id> -src_eid <src_eid> -dst_eid <dst_eid> [-pkt <size>] [-cnt <size>] [-interval <size>]
```
- 关键参数：
  - -i <dev_id> | 必选 | 软件的逻辑ID。执行npu-smi info命令查询到的“NPU ID”字段值即为-i参数取值。
  - -src_eid <src_eid> | 必选 | 指定源端EID标准格式，如x:x:x:x:x:x:x:x。可使用查询NPU设备信息命令查询，返回的字段“ipourma”的“IP Address”为可取的EID。
  - -dst_eid <dst_eid> | 必选 | 指定目的EID标准格式，如x:x:x:x:x:x:x:x。可使用查询NPU设备信息命令查询，返回的字段“ipourma”的“IP Address”为可取的EID。
  - -pkt <size> | 可选 | 指定ping的数据包大小。单位为Byte，取值范围为[0, 4046]，默认值为4046。
  - -cnt <size> | 可选 | 指定要发送的ping包数量。取值范围为[1, 1000]，默认值为3。
  - -interval <size> | 可选 | 指定两次ping包之间的发送时间间隔。单位为毫秒，取值范围为[0, 10000]，默认值为1000。
- 示例：
```bash
hccn_tool -t -ub_ping -i 0 -src_eid ::40:10:0:dfdf:90 -dst_eid ::60:10:0:dfdf:b0 -pkt 128 -cnt 6 -interval 1
```
